## 一、先把 ShowTheRate 的“工具入口”定义清楚

你现在的 3 个核心主题非常符合工具站逻辑：

## 二、找词方法：用“工具词公式”批量造词（这是核心）

你不要“想关键词”，你要“用公式生成关键词候选池”。

### 1）工具词四大公式（直接套用）

**公式 A：{对象} + calculator**

* `2-1 buydown calculator`
* `discount points calculator`
* `rate lock fee calculator`

**公式 B：{对象} + cost / fee / subsidy**（更贴近成交推进）

* `2-1 buydown cost` / `buydown subsidy`
* `rate lock extension fee`
* `points cost`

**公式 C：{对象} + break-even / worth it**（最强转化意图）

* `discount points break even`
* `float down worth it`
* `points worth it`

**公式 D：{对象} vs {对象}**（对比词就是“成交词”）

* `temporary buydown vs points`
* `lock vs float`
* `points vs lender credits`

> 这就是那篇文章讲的：**用户就想“算一下/对比一下”**，不是来读“科普”。

## 三、验证是不是“真工具词”：用 SERP 10 秒判断法

每个候选词，你都用 Google 搜一下（无痕窗口即可），只看一件事：

### ✅ 值得做成工具页的信号

### ❌ 不适合当主战场的信号

* SERP 明显是“科普意图”（how/what/guide），工具很少
* 你做工具也能做，但不应作为第一批 P0

> 这一步就是那篇文章里 “Ahrefs 看关键词排名/意图” 的替代版 —— 不用工具也能做“意图验证”。

## 五、落地方式：每个关键词必须对应“一个页面类型”

为了符合那篇文章的观点，你要强制把关键词映射成页面，而不是写文章：

### 1）Calculator 页（主战场）

关键词包含：`calculator / cost / fee / subsidy / break-even`

* 页面必须有可交互 mini-calculator
* 必须有默认示例（首屏就能看到结果）
* 必须有 CTA：Starter Pass / Pro

### 2）Compare 页（第二主战场）

关键词包含：`vs / compare / which is better`

* 对比表 + 结论区 + 适用人群
* 仍然要能算（至少两套方案结果）

### 3）Blog/Script 页（辅助，不是主战场）

关键词包含：`how to explain / script / what is`

* 文章必须内嵌对应 calculator（否则就是“博客思维”）
* 文章目的：给工具页输血 + 覆盖解释型流量

> 这就完全符合那篇文章的“工具词 vs 内容词”分工。
