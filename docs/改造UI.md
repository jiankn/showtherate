# UI改造方案（Landing + 后端App）

## 目标与原则（产品经理视角）
- 目标：提升转化率与信任度，缩短决策路径，提升后台任务效率与可追踪性
- 核心指标：注册/试用转化、CTA点击、内容阅读深度、工单首响时长、SLA风险工单占比、续费率
- 体验原则：数据优先、信息层级清晰、关键动作明确、反馈及时

## Landing Pages 改造（数据表现优先）

### 主页（Home）
- Hero：一句主价值 + 关键数据点（增长率/节省成本/转化提升）
- KPI条：3-5个核心指标 + 迷你趋势线
- 可信度：客户Logo + 结果型案例卡片（对比前后数据）
- 方案对比：计划/定价表 + 性能/安全/支持指标
- CTA：主CTA（开始试用）+ 次CTA（联系销售/看演示）

### Blog
- 顶部数据概览：阅读增长、热门主题占比、更新频率
- 文章卡片：阅读时间、热度指数、互动率
- 分类与筛选：基于数据标签（增长/留存/成本/安全）
- 研究报告模块：Benchmark/年度报告置顶

### Privacy Policy
- 数据处理总览：收集类型、用途、保留周期一览表
- 透明度模块：第三方共享清单 + 合规说明
- 版本记录：变更时间轴 + 生效日期

### Terms of Service
- TL;DR摘要：关键条款数据化（SLA、责任范围、退款条件）
- 服务等级：SLA指标 + 例外条件
- 变更机制：版本记录 + 通知机制说明

### 组件与动效
- 数据组件：KPI卡、趋势图、对比表、进度环
- 动效策略：数据计数渐入、图表描边加载、CTA微浮起
- 响应式：优先保证数据块在移动端的可读性与对比度

## 后端App页面改造方案（数据驱动）

### 信息架构策略
- 以“关键任务 + 风险提示 + 业绩指标”为主轴
- 导航分层：概览 / 收入 / 订阅 / 工单 / 设置
- 全局搜索与筛选：统一入口 + 保存视图

### 方案参考（选择其一为主方向）

#### 方案A：运营驾驶舱
- 适用：关注全局指标与风险提醒
- 特点：概览页重点展示KPI、SLA风险、收入趋势
- 主页结构：KPI行 + 趋势图 + 风险列表 + 任务队列

#### 方案B：工单优先
- 适用：客服效率与SLA是主目标
- 特点：工单队列置顶，支持快速分流、批量操作
- 主页结构：工单状态矩阵 + SLA预警 + 待处理列表

#### 方案C：增长优先
- 适用：营收与续费为核心
- 特点：订阅健康度、续费风险、ARPU变化可视化
- 主页结构：收入趋势 + 订阅分层 + 风险用户列表

## 页面级改造要点（建议）

### 概览（overview）
- KPI卡 + 7/30天趋势图
- 异常提醒：SLA风险、收入波动、续费下降
- 快捷任务：未读工单、待审批、异常订阅

### 收入（revenue）
- MRR/ARR折线 + 分解瀑布图
- 订阅流入/流出、退款/折扣影响
- 维度筛选：渠道/地区/计划

### 订阅（subscriptions）
- 健康评分与续费风险分层
- 关键用户列表（高价值/流失风险）
- 批量操作：提醒、优惠、跟进

### 工单（tickets）
- 队列优先级分层 + SLA倒计时
- 未读状态与处理阶段清晰可见
- 详情页：时间线、内部备注、客户上下文

### 设置（settings）
- 权限/角色配置、审计日志、通知偏好
- 关键安全项状态可视化（2FA、IP限制）

## 用户登录后App页面改造（面向LO）

### 目标与策略
- 目标：让LO更快生成对比、清晰解释方案、提升分享与跟进效率
- 策略：以“生成/分享/跟进”为主轴，数据只展示LO需要的成交信号

### 导航与全局体验
- 导航结构：Dashboard / Comparisons / Clients / Analytics / Support / Settings
- 全局能力：统一时间范围、快捷筛选、保存视图、最近操作入口
- 状态反馈：分享链接状态、客户打开反馈、配额与合规提示
- 行动优先：主CTA常驻（Generate / Share），移动端单手可达

### Dashboard（概览）
- 主区指标：本周对比生成数、分享链接数、客户打开数、待跟进数
- 效率指标：平均生成耗时、AI话术使用率、最常用方案模板
- 今日待办：未完成对比、即将过期分享、客户回复提醒
- 轻量趋势：近7天分享打开率/对比生成量（简洁折线即可）

### Comparisons（对比）
- 列表信息：状态（草稿/已分享/已打开/已过期）、最后编辑时间
- 快速动作：复制链接、重新分享、复制为新方案、生成话术
- 模板能力：保存为模板、常用模板置顶、快速新建
- 质量提示：缺失字段提醒（税费/保险/贷款类型）与一键补全引导

### Clients（客户）
- 视图结构：客户卡片包含状态（新线索/跟进中/已成交）、最近互动
- 关键数据：关联对比数量、最近打开的分享链接、备注与标签
- 行动入口：一键发送最新对比、发起跟进、记录客户异议
- 时间线：沟通记录 + 重要事件（看房/锁利率/到期提醒）

### Analytics（分析）
- 面向成交信号：分享打开率、客户回访率、对比被查看次数
- 受欢迎方案：最常被分享/被打开的对比模板
- 效率洞察：平均生成时间、方案修改次数、话术使用效果
- 轻量呈现：简洁趋势 + 排名卡片，避免复杂多维图表

### Support（支持）
- 快捷入口：话术模板库、异议处理脚本、FAQ
- 产品支持：工单提交、状态跟踪、紧急联系入口
- 合规提示：NMLS信息完整性检查、免责声明提示

### 交互与空态
- 空态：引导生成第一份对比、提供模板示例与最佳实践
- 复杂流程：分步填写、默认值策略、保存草稿与一键恢复

## 设计系统与规范
- 色彩：冷静蓝+青绿强调，风险使用橙/红
- 字体：标题用几何无衬线，数据用等宽或数字友好字体
- 组件：表格、标签、状态条、趋势图统一样式

## 实施步骤
1. 统一设计变量与数据组件库
2. Landing Pages 先行（主页为核心）
3. 后端概览与工单页面改造
4. 收入/订阅/设置模块落地
5. 动效与响应式细节打磨
